# BurstGridGo – Summary Report (July 2025)

## 1. Technical Critique of the PoC

| Area                     | Findings | Suggested 2025‑Level Improvement |
|--------------------------|----------|----------------------------------|
| **Project layout**       | `cmd/cli` exists, docs promise `cli`. | Pick one; keep Go convention (`cmd/burstgridgo`). |
| **Toolchain tags**       | No `//go:build` lines. | Add explicit build‑tags; drop legacy `+build`. |
| **Error handling**       | `log.Fatalf` inside goroutines aborts whole run. | Return errors, aggregate with `errgroup.Group`, exit once all workers finish. |
| **Context propagation**  | Runner API lacks `context.Context`. | `Run(ctx context.Context, mod Module)…` enables graceful cancel/timeouts. |
| **Executor robustness**  | No cycle detection; fixed worker pool of 4. | DFS cycle check; pool size `runtime.GOMAXPROCS()` or CLI flag. |
| **Generics use**         | Registry stores `Runner` (non‑typed), outputs as `cty.Value`. | `type Runner[T any]`, reduce runtime type‑switches. |
| **Observability**        | Uses `log.Printf`. | Switch to `log/slog`; ship OpenTelemetry spans per module. |
| **HTTP client**          | Creates new `http.Client` per request. | Reuse a singleton with custom `Transport`. |
| **Docker image**         | Distroless but CLI still exposes port 8080 & healthcheck. | Remove port/healthcheck or add real `/health` endpoint. |
| **Security toggle**      | `InsecureSkipVerify` boolean inside HCL. | Gated by `--allow‑insecure` CLI + prominent warning. |
| **Testing**              | Zero tests yet. | Start with DAG cycle, HCL decode, HTTP runner via `httptest`. |
| **CI/CD**                | No workflow file. | GH Actions: `go vet`, `go test -race`, image build + cosign attest. |

## 2. Concept Fit in 2025

* **Pain** Apps mix HTTP/3, WebTransport, Socket.IO, gRPC. Existing tools focus on HTTP or imperative JS DSLs.
* **Solution** Declarative HCL + auto‑derived DAG expresses *data flow*; Go runtime handles concurrency.
* **Why now**  
  * GitOps & IaC norm → HCL feels native to DevOps teams.  
  * Edge/CI runners need small static binaries.  
  * OTLP is commodity; a test tool that emits traces slots into Grafana stacks easily.

| **Managed replay SaaS** | k6 Cloud, Speedscale, Azure LT | N/A | Varies | Lock‑in / $$$. |

**Positioning:** BurstGridGo = IaC‑style config + any‑protocol runners + tiny static binary → sits between k6’s ecosystem and heavy SaaS solutions.
